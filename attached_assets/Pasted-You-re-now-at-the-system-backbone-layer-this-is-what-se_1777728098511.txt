# 🏢 9. MULTI-ROLE ENTERPRISE SYSTEM (Production Design)

## 9.1 Role Hierarchy & Permissions Model

### 🔷 Roles

| Role        | Scope                     | Power Level |
| ----------- | ------------------------- | ----------- |
| User        | Personal use              | Low         |
| Volunteer   | Respond to incidents      | Medium      |
| Admin       | Manage system (org-level) | High        |
| Authority   | Govt / emergency control  | Very High   |
| Super Admin | Platform-wide control     | Root        |

---

## 🔷 RBAC (Role-Based Access Control)

### Implementation Strategy

Use **Policy-Based RBAC (NOT hardcoded roles)**

```ts
permissions = {
  "incident:create": ["user", "volunteer"],
  "incident:assign": ["admin", "authority"],
  "incident:override": ["admin", "authority"],
  "system:manage": ["super_admin"]
}
```

---

### Middleware (Backend)

```ts
function authorize(action) {
  return (req, res, next) => {
    const role = req.user.role;
    if (!permissions[action].includes(role)) {
      return res.status(403).send("Forbidden");
    }
    next();
  };
}
```
