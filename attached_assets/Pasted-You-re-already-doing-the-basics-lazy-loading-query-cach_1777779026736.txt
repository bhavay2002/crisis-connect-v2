# 🧱 1. VIRTUALIZATION (Large Lists → Constant DOM Size)

## 🔷 Why it matters

Without virtualization:

* 1,000 items → 1,000 DOM nodes ❌
* slow scroll + layout thrashing

With virtualization:

* 1,000 items → ~20 DOM nodes ✅

---

## 🔷 Use `@tanstack/react-virtual` (best-in-class)

```tsx
import { useVirtualizer } from "@tanstack/react-virtual";
import { useRef } from "react";

export function CrisisList({ data }) {
  const parentRef = useRef<HTMLDivElement>(null);

  const rowVirtualizer = useVirtualizer({
    count: data.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 80,
    overscan: 5,
  });

  return (
    <div ref={parentRef} className="h-[600px] overflow-auto">
      <div
        style={{
          height: rowVirtualizer.getTotalSize(),
          position: "relative",
        }}
      >
        {rowVirtualizer.getVirtualItems().map((virtualRow) => {
          const item = data[virtualRow.index];

          return (
            <div
              key={item.id}
              style={{
                position: "absolute",
                top: 0,
                transform: `translateY(${virtualRow.start}px)`,
                width: "100%",
              }}
            >
              <CrisisCard crisis={item} />
            </div>
          );
        })}
      </div>
    </div>
  );
}
```

---

## 🔷 Industry Tips

* use `overscan: 5–10` for smooth scroll
* combine with **infinite loading**
* virtualize:

  * incident lists
  * chat messages
  * logs/timeline

# 🧠 2. MEMOIZED SELECTORS (Zustand / Redux)

## 🔷 Problem

```ts
const data = useStore((s) => s);
```

👉 re-renders on **any state change** ❌

---

## 🔷 Solution: Select + Shallow Compare

```ts
import { shallow } from "zustand/shallow";

const { reports, selectedId } = useStore(
  (s) => ({
    reports: s.reports,
    selectedId: s.selectedId,
  }),
  shallow
);
```

---

## 🔷 Advanced Pattern (Derived Selectors)

```ts
const criticalReports = useStore((s) =>
  s.reports.filter((r) => r.priority === "CRITICAL")
);
```

👉 BUT this recalculates every render ❌

---

### ✅ Fix with memoization

```ts
import { useMemo } from "react";

const reports = useStore((s) => s.reports);

const criticalReports = useMemo(
  () => reports.filter((r) => r.priority === "CRITICAL"),
  [reports]
);
```

---

## 🔥 Elite Pattern (Selector Outside Component)

```ts
const selectCritical = (s) =>
  s.reports.filter((r) => r.priority === "CRITICAL");

const criticalReports = useStore(selectCritical);
```

👉 cleaner + reusable

# 🎯 3. SELECTIVE RE-RENDER OPTIMIZATION

---

## 🔷 Problem

Parent re-renders → all children re-render ❌

---

## 🔷 Fix: `React.memo`

```tsx
export const CrisisCard = React.memo(function CrisisCard({ crisis }) {
  return <div>{crisis.title}</div>;
});
```

---

## 🔷 When to use

Use on:

* list items
* chat messages
* map markers

---

## 🔷 Custom comparison (advanced)

```tsx
export default React.memo(Component, (prev, next) => {
  return prev.crisis.id === next.crisis.id &&
         prev.crisis.status === next.crisis.status;
});
```

# 🔄 4. STABLE REFERENCES (HUGE IMPACT)

## 🔷 Problem

```tsx
<button onClick={() => handleClick(id)} />
```

👉 new function every render ❌

---

## 🔷 Fix

```tsx
const handleClick = useCallback((id) => {
  // logic
}, []);
```

---

## 🔷 Same for objects

```tsx
const style = useMemo(() => ({ color: "red" }), []);
```

# 🧵 5. REAL-TIME + PERFORMANCE (CRITICAL COMBO)

You have WebSockets → now optimize updates.

---

## 🔷 Problem

Incoming event → entire list updates ❌

---

## 🔷 Solution: Patch Updates

```ts
socket.on("CRISIS_UPDATE", (updated) => {
  set((state) => ({
    reports: state.reports.map((r) =>
      r.id === updated.id ? updated : r
    ),
  }));
});
```

---

👉 only 1 item updates → minimal re-render ✅

---

## 🔷 Even better: indexed store

```ts
reportsById: { [id]: report }
```
