## 🔷 5.1.1 User ↔ Volunteer Chat

### Data Model

```sql

- id
- incident_id
- participants (array)

- id
- conversation_id
- sender_id
- type (text/image/location)
- content
- created_at
- status (sent/delivered/read)
```

### Real-Time Flow

```text
User sends message

 → Backend validates
 → Store in DB
 → Publish via Redis
 → Deliver to receiver socket
```

### API + Events

```ts

SEND_MESSAGE
RECEIVE_MESSAGE
READ_RECEIPT
TYPING_START / STOP
```
