# 📦 3. COMPONENT SIZE SYSTEM

## 🔷 Problem

Buttons, cards, inputs → inconsistent ❌

---

## 🔷 Solution

Define size variants:

---

### Button System

```tsx id="button"
const sizes = {
  sm: "px-3 py-1 text-sm",
  md: "px-4 py-2",
  lg: "px-6 py-3 text-lg",
};

export function Button({ size = "md", children }) {
  return (
    <button className={`${sizes[size]} rounded-lg bg-primary text-white`}>
      {children}
    </button>
  );
}
```

---

### Card System

```tsx id="card"
export function Card({ children }) {
  return (
    <div className="bg-surface p-4 rounded-xl shadow-sm">
      {children}
    </div>
  );
}
```

---

👉 Everything now looks **designed, not assembled**

# 🎞️ 4. MOTION SYSTEM (HUGE DIFFERENTIATOR)

## 🔷 Problem

No motion → feels static ❌

---

## 🔷 Industry Rule

> Motion should guide attention, not decorate

---

## 🔷 Add Framer Motion

---

### Micro Interaction

```tsx id="hover"
<motion.button
  whileHover={{ scale: 1.03 }}
  whileTap={{ scale: 0.97 }}
>
  Dispatch
</motion.button>
```

---

### Entry Animation

```tsx id="fade"
<motion.div
  initial={{ opacity: 0, y: 10 }}
  animate={{ opacity: 1, y: 0 }}
  transition={{ duration: 0.2 }}
>
```

---

### Real-Time Pulse

```tsx id="pulse"
<motion.div
  animate={{ opacity: [1, 0.5, 1] }}
  transition={{ repeat: Infinity, duration: 1 }}
>
  CRITICAL
</motion.div>
```

---

## 🔷 Motion Rules

| Action | Animation |
| ------ | --------- |
| hover  | scale     |
| update | highlight |
| alert  | pulse     |
| load   | fade      |
