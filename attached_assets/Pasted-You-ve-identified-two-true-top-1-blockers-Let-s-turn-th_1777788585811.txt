## 🔷 Step 1: Fast Ingestion API (Non-blocking)

```ts
// POST /reports
export async function createReport(req, res) {
  const report = await db.reports.create({
    data: {
      text: req.body.text,
      status: "PENDING_AI",
    },
  });

  await queue.add("ai-analysis", { reportId: report.id });

  return res.status(202).json({
    message: "Report received",
    reportId: report.id,
  });
```

## 🔷 Step 3: AI Worker (Decoupled Service)

```ts
// worker.ts
import { Worker } from "bullmq";

const worker = new Worker("ai-jobs", async (job) => {
  const { reportId } = job.data;

  const report = await db.reports.findById(reportId);
```

# ⚡ 2. WEBSOCKET SCALING — PUB/SUB ARCHITECTURE

## 🔷 Problem Recap

```text
Single Node
→ holds connections in memory ❌
→ breaks on scale
```

---

## 🔷 Target Architecture

```text
Client
  ↓
Multiple Node Instances
  ↓
Redis Pub/Sub
  ↓
All instances sync events
```

---

## 🔷 Step 1: Add Redis Adapter

Use:

👉 **Socket.IO Redis Adapter**

```ts
import { createAdapter } from "@socket.io/redis-adapter";
import { createClient } from "redis";

const pubClient = createClient();
const subClient = pubClient.duplicate();

await pubClient.connect();
await subClient.connect();

io.adapter(createAdapter(pubClient, subClient));
```

---

## 🔷 Step 2: Publish Events from Worker

```ts
pubClient.publish(
  "report.updated",
  JSON.stringify({ reportId, aiResult })
);
```

---

## 🔷 Step 3: Subscribe in Each Instance

```ts
subClient.subscribe("report.updated", (message) => {
  const data = JSON.parse(message);

  io.emit("REPORT_UPDATE", data);
});
```

---

## 🔷 Step 4: Room-Based Optimization

```ts
io.to(`incident:${incidentId}`).emit("UPDATE", data);
```

👉 avoids broadcasting to all clients
