## 🔷 Option A — Redis Streams (fast to ship)

### Create & publish

```ts id="rs_pub"
// publish
await redis.xAdd("events:reports", "*", {
  type: "report.created",
  reportId,
  ts: Date.now().toString()
});
```

### Consumer group (durable + retry)

```ts id="rs_consume"
await redis.xGroupCreate("events:reports", "ai-workers", "$", { MKSTREAM: true });

while (true) {
  const res = await redis.xReadGroup(
    "ai-workers",
    "worker-1",
    [{ key: "events:reports", id: ">" }],
    { COUNT: 10, BLOCK: 5000 }
  );

  for (const msg of parse(res)) {
    try {
      await handle(msg);
      await redis.xAck("events:reports", "ai-workers", msg.id);
    } catch (e) {
      // no ack → will be retried
    }
  }
}
```

### Replay (key advantage)

```ts id="rs_replay"
// read from a past ID or time range
await redis.xRange("events:reports", "1700000000000-0", "+");
```

# 📡 9) EXTERNAL API RESILIENCE (BEYOND CIRCUIT BREAKER)

## 🔷 Add a “Resilience Layer”

```text id="res_layer"
Service → Cache → Fallback Dataset → External API
```

---

## 🔷 1. Cache + TTL (first line)

```ts id="cache1"
const key = `weather:${lat}:${lng}`;
const cached = await redis.get(key);

if (cached) return JSON.parse(cached);

const fresh = await fetchWeather();
await redis.set(key, JSON.stringify(fresh), { EX: 300 }); // 5 min TTL
return fresh;
```

---

## 🔷 2. Fallback Dataset (second line)

Keep a **local, last-known snapshot** (DB or object storage).

```ts id="fallback1"
try {
  return await fetchWeather();
} catch {
  const last = await db.weatherSnapshots.findLatest(lat, lng);
  return { ...last, stale: true };
}
```

---

## 🔷 3. Cached Predictions (third line)

Store last good inference for reuse.

```ts id="pred_cache"
const key = `pred:${area}`;
const pred = await redis.get(key);

if (pred) return { ...JSON.parse(pred), stale: true };
```

---

## 🔷 4. Degraded Mode UI (don’t hide it)

```text id="ui_degraded"
⚠️ Live data unavailable
Using last known update (5 min old)
```

---

## 🔷 5. Bulkhead isolation

* Separate clients/pools per dependency
* Failure in weather API ≠ blocks AI or dispatch

---

## 🔷 Result

* system continues with **“stale but usable”** data
* no cascading failure from third parties
